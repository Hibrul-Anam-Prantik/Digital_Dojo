# 🏍️ CYBER RIDER // Neon Highway

An action-packed, retro-synthwave neon motorcycle racing game built directly in the browser. Dodge barriers, collect energy batteries, and outrun the neural net CPU lightcycle in a high-speed race across the cyber grid.

---

## 🎮 Game Concept & Objective

In **Cyber Rider**, you take control of a neon lightcycle navigating a shifting cyber highway. 
- **The Goal**: Reach the 2000m finish line before the CPU-controlled opponent bypasses the neural network.
- **Hazards**: Yellow energy barriers that crash your bike and drastically slow you down.
- **Power-ups**: Pink battery cells that recharge your Nitro speed boost.

---

## 🛠️ Tech Stack

This project was built from scratch without external libraries, using:

*   **HTML5 & Canvas API**: Used for 60 FPS frame rendering, parallax scrolling backdrops, dynamic trail drawings, and particle explosion FX.
*   **Vanilla CSS3 (OKLCH)**: Designed with modern OKLCH color palettes, custom neon glow effects, glassmorphic HUD cards, scanline CRT screen overlays, and responsive design systems.
*   **Vanilla JavaScript (ES6+)**: Powers the game physics engine, AI steering logic, and collision detection.
*   **Web Audio API**: Dynamically synthesizes real-time engine hums (pitch-shifting based on current speeds), crash explosions, and boost sound FX directly from code without audio files.

| Control | Action |

| **▲ / W** | Switch to the Upper Lane |
| **▼ / S** | Switch to the Lower Lane |

## 🚀 Built with Google Antigravity (AGY) CLI

This game was built and optimized as part of a learning journey exploring the capabilities of **Google Antigravity (AGY) CLI**. 

Throughout the process, the project was developed by pairing up with the Antigravity AI coding agent to:
- Learn and practice AGY CLI commands and execution permissions.
- Manage Git versions and commit states from the command-line interface.
- Solve responsive layout sizing constraints (fitting the canvas layout dynamically across screen heights).
- Debug core JS initialization issues (re-ordering constructor configuration parameters to fix coordinate scaling bugs).
